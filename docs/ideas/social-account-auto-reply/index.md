# 多平台账号自动回复

面向抖音、快手、小红书等**账号侧私信 / 评论等消息**的自动回复项目：**Pydantic AI 后端**（记忆体为主）+ **浏览器扩展**（页面执行）。合规与平台策略在落地产品中单独设计，此处只作技术与产品备忘。

## 一句话协作关系（会随架构文档细化）

扩展与后端保持 **WebSocket（WSS）长连接**：扩展**上报**观测与工具回执，后端在服务端跑 agent 循环并**下发**调用过程（多步 tool、顺序与参数）；扩展按**本地「类 API」**执行。细节见 [架构与数据流](./architecture/index.md) 与 [通信与协议](./protocol/index.md)。

## 共通风险（各子区仍会展开）

- 后端 tool 与前端函数**契约**要同步维护。  
- 各站 DOM 易变，适配宜可配置、分版本。  
- 鉴权、频率、人工确认与平台规则要在产品层显式设计。
