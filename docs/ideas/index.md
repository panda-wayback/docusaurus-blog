# 想法与项目

**日常随想**与**准备立项、写清技术骨架的项目**，放在**同一顶栏分区**里：是一条连续线——碎片记录加深之后，自然长成子文件夹里的「项目文档」，不必再拆成两个入口，避免割裂。

## 这条线怎么理解

- **轻量**：随手记、小 demo、一闪而过的念头；可以很短、不成体系。  
- **加重**：当某条想法值得展开架构、协议、变现、实现细节时，为它建**子文件夹**（例如下面的「多平台账号自动回复」），在同一分区里从随笔**长成项目**。  
- **目录习惯**：大区 `index.md` 做汇总；具体功能或深写再拆子夹（见仓库 `.cursor/skills` 里文档规范）。

## 分层与约定

- 可单开 `.md`，或按主题建文件夹；项目内部再按主题拆 **架构 / 协议 / 前后端 / 待决** 等——**名字随内容**，不必背固定清单。  
- 未定稿标「占位」或写在待决区，定稿后再收紧表述。
