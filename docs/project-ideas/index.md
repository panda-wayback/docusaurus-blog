# 项目想法

这里记录**打算认真推进**的方向：相对「日常想法」，会更明确**要解决谁的问题、怎么收钱、技术怎么搭骨架**，方便当作项目备忘录和立项草稿。

## 目录约定

- **每个独立项目一个子文件夹**（例如下面的「多平台账号自动回复」），避免多个设想混在同一篇长文里。  
- 项目内部再按 **架构 / 协议 / 后端 / 前端 / 产品 / 待决** 等子文件夹拆分，侧栏顺序由各自 `_category_.json` 的 `position` 固定。  
- 未定稿内容在对应区标注「占位」或写在 **待决与备忘**，定稿后迁到正式页。
