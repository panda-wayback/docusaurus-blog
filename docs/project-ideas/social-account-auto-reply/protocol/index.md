# 通信与协议

> **传输层已定：WebSocket（WSS）**。扩展主动连后端；消息字段与版本化仍待补全。

## 传输层

- **WebSocket over TLS（WSS）**：一条（或按需在协议中规定的少数几条）长连接承载会话。  
- **建立方式**：由**浏览器扩展**发起连接；后端接受后可在任意时刻**推送**调用指令，扩展也可随时**上报**（无需为「后端驱动前端」再单独做扩展侧 HTTP 服务）。  
- **与轮询**：同一会话内的指令与回执走 WS **推送**，一般不另用 HTTP 轮询承载同一类实时编排（除非降级或运维探测等例外，另议）。

## 计划在本文档写清的内容（待补）

- **扩展 → 后端**：连接建立后的鉴权、会话 id；事件类型（新消息、心跳、错误）；观测载荷字段。  
- **后端 → 扩展**：下发「调用过程」的 JSON 形状（工具名、参数、顺序、call id、是否可并行等）。  
- **扩展 → 后端**：每步执行**回执**（成功 / 失败、错误码、业务 id；敏感 DOM 是否截断上传等策略）。  
- **鉴权**：令牌放首帧 / 子协议 / 首条消息；刷新策略；是否与扩展 id / 设备指纹绑定（若需要）。
