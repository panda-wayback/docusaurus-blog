# 新消息上行

## 触发

- 扩展识别「**某人向我发来消息**」：DOM 变化、会话列表高亮、未读角标，或站点内部消息通道（若可稳定订阅）。

## 上报（经 WebSocket）

- 建议载荷包含：**平台**、**会话 id**、**对方用户标识**、**消息正文或摘要**、**时间戳**、**是否首条未读**等。  
- 细字段与版本化在 [通信与协议](../../protocol/index.md) 定稿。

## 与后端的衔接

- 后端据此做消息流水与用户画像，见 [用户消息流水与画像](../../backend/user-chat-profile/index.md)。  
- 智能体回复策略依赖上述数据，见 [记忆体](../../backend/memory/index.md) 与 [Tool 与契约](../../backend/tools-contract/index.md)。
