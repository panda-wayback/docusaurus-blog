# 前端与扩展

## 「类 API」的意图

- 扩展内实现**一组原子能力**（例如：向某会话发送文本、读取当前列表、切换会话、滚动未读），每个能力有**稳定标识与参数约定**，集中注册，类似**对内 API 表**。  
- 好处：后端下发的「调用过程」只需映射到这张表；换站点适配时尽量**只改实现、少改协议**。  

## 与后端的配合

- 与后端维持 **WebSocket（WSS）**（见 [通信与协议](../protocol/index.md)）：监听页面、解析 DOM 或站点前端接口（若有）后，经连接把**观测结果**发给后端。  
- 收到后端经同连接下发的 payload 后，**派发**到对应本地函数执行；将**回执**再经连接回传，供 [后端（Pydantic AI）](../backend/index.md) 下一轮 agent 循环使用。  

## 实现备忘（非规范）

- Manifest V3、service worker 与 content script 消息通道；**具体文件结构以后单开文档再写**，避免和协议、架构混在同一页。  
- 各站 DOM 脆弱：选择器配置化、分版本维护；优先官方开放能力（若有）。
