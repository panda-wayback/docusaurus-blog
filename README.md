# Panda 的技术博客（Docusaurus）

基于 [Docusaurus](https://docusaurus.io/) 的静态文档站。本文档既说明**本仓库在做什么**，也作为**把同样模式搬到其它 Docusaurus 项目**时的操作清单（新建站点或改造模板均可按「迁移步骤」执行）。

---

## 本仓库相对官方模板的定位

官方 `classic` 模板通常约定：

- `docs/` 下是一份连续文档树；
- **顶栏**只有一个「文档」入口，指向**单一**侧栏（`sidebars.ts` 里手写 `tutorialSidebar` 等）；
- 若要「多个文档分区、每个分区一套侧栏」，需要自己维护多份侧栏并在 `navbar.items` 里写多个 `docSidebar`，**每加一个分区都要改配置**。

本仓库的**核心改动**是：用 **`docs-nav-autogen.ts`** 在构建时扫描 `docs/` 的**一级子目录**，为每个目录**自动生成**：

1. **顶栏**：一项 `type: 'docSidebar'`，标签文字可配置（见下文 `_category_.yml`；本站 `docs/` 已全部使用该格式）；
2. **侧栏**：侧栏 ID 为 `<一级目录名>Sidebar`，内容为该目录下的 **`autogenerated`** 树。

这样 **新增/重命名一级目录** 时，只要动文件系统，**不必**再手改 `sidebars.ts` 和顶栏数组（除非你要改排序、插入自定义链接等）。

---

## 行为说明（读代码前先看这段）

### 顶栏与侧栏如何对应

- Docusaurus 里，`navbar.items` 中的 `{ type: 'docSidebar', sidebarId: 'xxxSidebar', label: '...' }` 表示：点击顶栏该项时，左侧文档侧栏切换为 ID 为 `xxxSidebar` 的那套配置。
- `sidebars.ts` 里导出 `SidebarsConfig`，键名就是侧栏 ID（例如 `financeSidebar`）。
- 本仓库规则：**一级目录名** `my-topic` → 侧栏 ID **`my-topicSidebar`**（与文件夹名一致，含连字符等）。

### 文档放在哪里

- **必须**放在某个一级目录下，例如 `docs/tutorial/intro.md`，路由一般为 `/docs/tutorial/intro`（与 `slug` / 文件路径有关）。
- **`docs/` 根目录**若放 `intro.md`，**不会**自动进某个「分区侧栏」，但本仓库用 `navbarIntroItem()` 在顶栏加 **「简介」** 链到 `/docs/intro`（存在 `intro.md` 或 `intro.mdx` 时）。

### 哪些目录会被扫描

- 只扫描 **`docs/` 下直接子目录**。
- **排除**名称以 `_` 或 `.` 开头的目录（与 Docusaurus 常见约定一致，避免工具目录进顶栏）。

### 顶栏上显示的名字从哪来

对每个一级目录 `folder`：

1. 若存在 **`docs/<folder>/_category_.yml`** 且含 **`label`** → 顶栏优先用该字符串（推荐：YAML 可写 `#` 中文注释，见仓库 `config-examples/_category_.yml` 样例）。
2. 否则若存在 **`_category_.json`**（Docusaurus 仍支持；`docs-nav-autogen` 亦作回退）且含 **`label`** → 顶栏用该字符串。
3. 否则仍会为该目录生成顶栏项，`label` 由文件夹名推导（将 `-`/`_` 等拆词并做英文式首字母大写，例如 `panda-blog` → `Panda Blog`）。

子目录里的 `_category_.yml` 仍按 **Docusaurus 官方语义** 影响**侧栏里该层分类**的显示；顶栏只读取**一级目录**的 `label`，逻辑独立。

### 分类配置文件与注释

- Docusaurus 官方支持 **`_category_.yml`**，与 `.json` 字段相同；**YAML 可写注释**。本站 **`docs/` 下已全部使用 `.yml`**。
- 仓库根目录 **`config-examples/_category_.yml`**：各字段均有中文说明，新建分区时可复制再改。
- **`docs-nav-autogen.ts`** 为读取 YAML 中的 `label`，依赖 npm 包 **`js-yaml`**（见 `package.json`）；迁移到其它项目时请执行 `yarn add js-yaml`。

### 排序

- 一级目录在顶栏中的顺序：当前实现为 **按目录名字符串排序**（`localeCompare`，`zh-Hans`）。若要完全自定义顺序，需要改 `docs-nav-autogen.ts`（例如读 `position`）或改回手写 `navbar.items`。

## 迁移到其它 Docusaurus 项目（逐步操作）

以下假设目标项目已能 `yarn start` / `yarn build`，且使用 **TypeScript 配置**（`docusaurus.config.ts`）与 **classic 预设**。

### 1. 复制核心脚本

将本仓库根目录的 **`docs-nav-autogen.ts`** 复制到目标项目**根目录**（与 `docusaurus.config.ts` 同级）。

需安装依赖 **`js-yaml`**（用于解析 `_category_.yml` 中的 `label`）：`yarn add js-yaml`。

### 2. 修改 `sidebars.ts`

用自动生成的侧栏**展开**替换模板里手写侧栏（或保留手写名并合并，但多分区场景通常整段替换更省事）。

**最小形态**（与本仓库一致）：

```ts
import type {SidebarsConfig} from '@docusaurus/plugin-content-docs';
import {buildSidebarsFromDocFolders} from './docs-nav-autogen';

const sidebars: SidebarsConfig = {
  ...buildSidebarsFromDocFolders(),
};

export default sidebars;
```

含义：`buildSidebarsFromDocFolders()` 为每个 `docs/` 一级目录生成一项 `{ type: 'autogenerated', dirName: '<一级目录名>' }`，侧栏 ID 为 `<一级目录名>Sidebar`。

### 3. 修改 `docusaurus.config.ts`

#### 3.1 在文件顶部增加导入

```ts
import {
  buildNavbarDocSidebarItems,
  navbarIntroItem,
} from './docs-nav-autogen';
```

#### 3.2 配置 `themeConfig.navbar.items`

在顶栏中插入「简介」（可选）与「各分区文档」项，并保留搜索等。典型结构：

```ts
navbar: {
  // title / logo 等略
  items: [
    ...(navbarIntroItem() ? [navbarIntroItem()!] : []),
    ...buildNavbarDocSidebarItems(),
    { type: 'search', position: 'right' },
  ],
},
```

说明：

- **`navbarIntroItem()`**：若存在 `docs/intro.md` 或 `docs/intro.mdx`，返回 `{ to: '/docs/intro', label: '简介', position: 'left' }`，否则返回 `null`。不需要「简介」时，可删除 `intro.md` 并去掉相关展开，或改 `docs-nav-autogen.ts` 中的逻辑。
- **`buildNavbarDocSidebarItems()`**：返回与 `sidebars.ts` 中各 `*Sidebar` ID 一一对应的 `docSidebar` 项。
- 模板自带的 **`type: 'doc', docId: '...'`** 单文档入口若与多分区冲突，应删掉，避免侧栏与路由混乱。

#### 3.3 确认 `preset-classic` 中文档插件

```ts
docs: {
  sidebarPath: './sidebars.ts',
  // editUrl 建议改为你的仓库地址；不需要可设为 undefined
},
```

### 4. 调整 `docs/` 目录结构

- 为每个希望出现在顶栏的「分区」建 **一级子目录**，例如 `docs/tutorial/`、`docs/blog-config/`。
- 每个分区下按 Docusaurus 习惯放 `*.md` / `*.mdx`；需要侧栏分类名、折叠行为时，在对应目录放 **`_category_.yml`**（[官方说明](https://docusaurus.io/docs/sidebar/autogenerated#category-item-metadata)）。
- 可选：在 `docs/` 根放 **`intro.md`**，配合顶栏「简介」。

**注意**：若某个一级目录下没有任何可索引文档，可能出现空侧栏或构建警告，按官方文档补齐至少一篇或删除该目录。

### 5. 关闭博客（若不用 `blog/`）

与模板默认不同，本仓库未使用博客目录，在 `preset-classic` 中设置：

```ts
blog: false,
```

否则构建可能仍尝试加载不存在的博客模块。若你**需要博客**，应保留 `blog/` 或配置独立博客插件路径，**不要**简单设为 `false`。

### 6. 本地搜索（可选，与本仓库一致时）

1. 安装依赖：

   ```bash
   yarn add @easyops-cn/docusaurus-search-local
   ```

2. 在 `docusaurus.config.ts` 的 `plugins` 数组中加入（选项可按插件文档微调）：

   ```ts
   [
     require.resolve('@easyops-cn/docusaurus-search-local'),
     {
       hashed: true,
       language: ['en', 'zh'],
       highlightSearchTermsOnTargetPage: true,
       explicitSearchResultPath: true,
       indexBlog: false, // 未开博客时
     },
   ],
   ```

3. 顶栏保留 `{ type: 'search', position: 'right' }`（与 classic 主题兼容）。

若你使用 **Algolia**，可不用该插件，而在 `themeConfig` 中配置 Algolia；二者择一即可，避免重复搜索 UI。

### 7. 国际化与站点 URL（按需）

- 本仓库 `i18n.defaultLocale` 为 **`zh-Hans`**，可按目标站点修改。
- GitHub Pages **项目站** 通常 `baseUrl` 为 **`/<仓库名>/`**，`url` 为 **`https://<用户或组织>.github.io`**；**用户/组织主页** 仓库则常为 `baseUrl: '/'`。新建项目后务必改 `url`、`baseUrl`、`organizationName`、`projectName`，否则资源路径与链接会错。

### 8. 类型检查（可选）

根目录 `tsconfig.json` 使用 `"extends": "@docusaurus/tsconfig"` 时，需在 **`devDependencies`** 中安装 **`@docusaurus/tsconfig`**，与 `@docusaurus/core` 主版本对齐。`package.json` 中已有 `"typecheck": "tsc"` 时可运行 `yarn typecheck`。

## 本仓库其它常见配置（非必须，但迁移时可对照）

| 项目 | 说明 |
|------|------|
| `onBrokenLinks: 'throw'` | 构建遇断链直接失败，便于 CI 拦截坏链接。 |
| `future.v4: true` | 提前对齐 Docusaurus v4 行为，可按官方文档决定是否开启。 |
| `trailingSlash: false` | URL 是否带尾部斜杠，需与托管/反向代理规则一致。 |
| Footer / `editUrl` | 本仓库仍部分保留模板链接，迁移新项目时应改成真实仓库与页脚内容。 |

## 涉及文件一览（迁移时重点看这几处）

| 路径 | 作用 |
|------|------|
| `docs-nav-autogen.ts` | 扫描 `docs/` 一级目录，生成侧栏配置与顶栏 `docSidebar` 项；可选「简介」顶栏项。 |
| `sidebars.ts` | 导出 `buildSidebarsFromDocFolders()` 的展开结果。 |
| `docusaurus.config.ts` | 引入上述函数，组装 `navbar.items`；站点 URL、插件、预设等。 |
| `docs/<分区>/` | 各分区文档与 **`_category_.yml`**（本站统一；亦可使用官方支持的 `.json`）。 |
| `config-examples/_category_.yml` | 分类元数据样例（各字段中文注释），可复制到 `docs/` 下使用。 |
| `docs/intro.md` | 可选全站「简介」入口。 |


## 部署提示

使用 **GitHub Pages** 时，将 `url`、`baseUrl`、`organizationName`、`projectName` 设为实际仓库与托管方式；执行部署前阅读 [Docusaurus 部署文档](https://docusaurus.io/docs/deployment)。`yarn deploy` 会依赖仓库权限与分支策略，以官方说明为准。
